# 语言教学预约系统软件设计说明文档

# 一、文档引言

## 1.1 文档目的

本文档基于《语言教学预约系统需求分析文档》，详细阐述系统的API设计和前端设计方案，明确各模块的实现逻辑、接口定义、页面布局及交互规则，为系统开发、测试、部署及维护提供标准化依据，确保开发过程贴合需求、高效推进，最终实现系统既定目标。

## 1.2 设计范围

本次设计覆盖系统全部核心功能，分为API设计和前端设计两大模块：

- API设计：涵盖用户注册与认证、课程与排期管理、学生预约、课程执行与后续流程、管理员后台管理等所有业务接口，明确接口地址、请求方式、参数说明、返回格式及异常处理。

- 前端设计：涵盖学生端、教师端、管理员端的所有页面布局、交互逻辑、组件设计、路由配置及适配要求，确保操作便捷、体验流畅，符合可用性需求。

## 1.3 设计原则

- 实用性：严格贴合需求文档，确保设计方案可落地、能实现所有功能需求，满足学生、教师、管理员的实际使用场景。

- 安全性：遵循需求中的安全要求，API层实现权限控制、数据加密，前端层防范常见攻击，保障用户信息和系统数据安全。

- 可扩展性：设计预留扩展接口和组件，支持后续新增语言类型、课程形式、支付方式等功能，降低迭代成本。

- 易用性：前端设计遵循简洁明了的操作逻辑，API设计接口规范、易懂，便于开发对接和后期维护。

- 高效性：优化API响应速度和前端页面加载性能，满足性能需求中“1000人同时在线、页面加载≤2秒”的要求。

# 二、API设计

## 2.1 设计概述

API设计采用RESTful风格，基于HTTP/HTTPS协议进行通信，统一请求/返回格式，采用JSON作为数据交换格式。接口按业务模块分类，实现角色权限隔离（基于RBAC模型），确保不同角色只能访问对应权限的接口。同时，所有接口均实现数据校验、异常处理和日志记录，保障接口稳定性和安全性。

基础配置：

- 接口基础路径：/api/v1

- 请求方式：GET（查询）、POST（新增）、PUT（修改）、DELETE（删除）

- 请求头：Content-Type: application/json；Authorization: Bearer {token}（登录后接口必传）

- 返回格式：统一包含code（状态码）、message（提示信息）、data（返回数据，可选）

- 状态码规范：200（成功）、400（参数错误）、401（未登录/Token失效）、403（权限不足）、404（资源不存在）、500（服务器异常）

## 2.3 安全设计

- 数据加密：所有敏感参数（密码、支付信息）传输时采用HTTPS加密，密码存储采用bcrypt算法+随机盐值，支付信息采用AES加密存储。

- 权限控制：基于RBAC模型，每个接口都进行角色权限校验，防止越权访问；Token采用JWT加密，定期刷新，失效后需重新登录。

- 接口防护：所有接口实现参数校验，防范SQL注入（采用参数化查询）、XSS攻击（输入过滤转义）；接口调用频率限制，防止恶意请求。

- 日志记录：所有接口调用、异常信息、敏感操作（登录、支付、退款）均记录日志，便于排查问题和追溯操作。

## 2.4 异常处理

统一异常处理机制，针对不同异常场景返回对应状态码和提示信息，主要异常类型包括：

- 参数异常：请求参数缺失、格式错误，返回400状态码，提示具体错误信息（如“手机号格式不正确”）。

- 权限异常：未登录、Token失效返回401，权限不足返回403，提示“请先登录”“权限不足，无法操作”。

- 资源异常：请求的课程、订单、用户不存在，返回404，提示“资源不存在”。

- 业务异常：如学生超时取消预约、教师未审核就创建课程等，返回400，提示具体业务限制（如“已超过取消预约时间，无法取消”）。

- 服务器异常：系统内部错误，返回500，提示“服务器繁忙，请稍后再试”，同时记录详细异常日志。

# 三、前端设计

## 3.1 设计概述

前端采用前后端分离架构，基于Vue3+Element Plus开发，支持PC端和移动端适配（响应式设计），确保操作流程简洁，学生完成预约操作不超过3步。前端分为学生端、教师端、管理员端三个子系统，共享基础组件和工具类，各自实现对应角色的功能，同时遵循统一的设计规范，保证界面风格一致、交互流畅。

核心设计规范：

- 技术栈：Vue3、Vue Router、Pinia（状态管理）、Element Plus（UI组件）、Axios（接口请求）、Vite（构建工具）。

- 适配要求：PC端支持1920*1080、1366*768等主流分辨率；移动端支持iOS、Android主流机型，适配不同屏幕尺寸。

- 交互规范：按钮点击反馈、表单验证实时提示、加载状态显示、分页加载（数据量较大时），确保操作反馈及时。

- 安全规范：前端不存储敏感信息（密码、支付信息），所有接口请求携带Token，敏感数据展示时进行脱敏（如手机号隐藏中间4位）。

## 3.2 整体架构设计

### 3.2.1 目录结构

```plain text
src/
├── assets/          # 静态资源（图片、样式、图标）
├── components/      # 公共组件（按钮、表单、弹窗等）
├── views/           # 页面视图（按角色分目录）
│   ├── student/     # 学生端页面
│   ├── teacher/     # 教师端页面
│   └── admin/       # 管理员端页面
├── router/          # 路由配置（分角色路由，权限控制）
├── store/           # 状态管理（用户信息、全局状态）
├── api/             # 接口请求封装（按模块分文件）
├── utils/           # 工具类（日期处理、加密、权限判断等）
├── hooks/           # 自定义钩子（复用逻辑，如表单验证、接口请求）
└── App.vue          # 根组件
```

### 3.2.2 路由设计

采用路由懒加载，实现角色权限路由控制，未登录用户只能访问登录、注册页面；不同角色只能访问对应权限的路由，越权访问将跳转至403页面。

核心路由配置示例：

- 公共路由：/login（登录）、/register/student（学生注册）、/register/teacher（教师注册）、/forgot-password（密码找回）。

- 学生端路由：/student/home（首页）、/student/course（课程筛选）、/student/order（我的订单）、/student/evaluate（我的评价）。

- 教师端路由：/teacher/home（首页）、/teacher/course（我的课程）、/teacher/schedule（课程排期）、/teacher/record（课程记录）。

- 管理员端路由：/admin/home（首页）、/admin/teacher-audit（教师审核）、/admin/course-manage（课程管理）、/admin/data-statistic（数据统计）。

### 3.2.3 状态管理设计

使用Pinia管理全局状态，主要分为以下模块：

- userStore：存储用户信息（用户ID、角色、姓名、手机号等）、登录状态、Token，提供登录、退出、更新用户信息的方法。

- globalStore：存储全局配置（如系统标题、主题色）、加载状态、提示信息等公共状态。

- orderStore：存储学生预约订单相关状态（待支付订单、已预约订单等），供学生端相关页面复用。

## 3.3 各端页面设计

### 3.3.1 公共页面设计

所有端共享的基础页面，风格统一，实现核心公共功能：

- 登录页面：支持手机号/邮箱登录，密码隐藏显示，忘记密码跳转，第三方登录入口（可选），表单验证（账号、密码必填，格式校验）。

- 注册页面：分学生/教师注册，表单按需求设计，实时验证（手机号格式、密码强度、邮箱格式），提交后提示注册结果。

- 密码找回页面：输入账号（手机号/邮箱），获取验证码，验证通过后重置密码，表单验证确保两次密码一致。

- 403/404/500页面：异常页面，提供返回首页按钮，提示具体异常信息，引导用户操作。

### 3.3.2 学生端页面设计

核心目标：便捷完成课程预约、查看订单、评价教师，操作流程简洁，符合学生使用场景。

- 首页：展示推荐课程、热门教师、个人预约提醒（即将开始的课程），提供课程筛选入口、个人中心入口。

- 课程筛选页面：提供多条件筛选（语言类型、难度等级、课程形式、教师类型），筛选结果分页展示，课程卡片显示核心信息（课程名称、教师、课时费、难度），点击可查看课程详情。

- 课程详情页面：展示课程详情（教学内容、课程特色）、教师信息（资质、评分、评价）、可预约时间段（日历视图，已预约时段标灰），提供“立即预约”按钮，跳转至预约确认页面。

- 预约确认页面：展示预约信息（课程、时间、费用），确认无误后提交，生成待支付订单，跳转至支付页面。

- 支付页面：支持多种支付方式（余额、微信、支付宝），展示支付金额、订单信息，支付完成后跳转至订单详情页面，提示预约成功。

- 我的订单页面：按订单状态（待支付、已预约、已完成、已取消）分类展示，可查看订单详情、取消预约（规定时间内）、支付待支付订单。

- 评价页面：课程结束后，收到评价提醒，跳转至评价页面，可对教师进行评分（1-5星）和文字评价，提交后展示评价结果。

- 个人中心：展示个人信息（可编辑）、学习目标、语言基础、余额，提供密码修改、退出登录功能。

### 3.3.3 教师端页面设计

核心目标：高效管理课程、排期、订单，提交课程记录，处理预约相关事务。

- 首页：展示个人课程统计（已创建课程数、今日预约数、待上课程），提供课程创建、排期设置入口，展示系统通知（如学生预约、取消预约）。

- 我的课程页面：展示已创建的所有课程，可编辑课程详情、删除课程，查看课程预约记录。

## 3.6 安全与性能优化

### 3.6.1 安全优化

- 敏感信息脱敏：展示手机号、邮箱时，隐藏中间部分（如手机号138****8888），不展示完整敏感信息。

- Token管理：登录后Token存储在localStorage，设置过期时间，退出登录时清除Token；接口请求时自动携带Token，Token失效时跳转至登录页面。

- 输入过滤：对用户输入的文本进行过滤，转义特殊字符，防范XSS攻击；表单提交前校验参数，避免恶意输入。

### 3.6.2 性能优化

- 路由懒加载：减少初始加载资源，提高页面加载速度，确保页面加载时间≤2秒。

- 图片优化：静态图片压缩处理，使用懒加载（图片进入视口后加载），减少加载耗时。

- 接口请求优化：封装Axios，实现请求拦截、响应拦截，统一处理Token、异常；数据量大时采用分页加载、下拉刷新，避免一次性加载过多数据。

- 缓存优化：对常用数据（如课程模板、用户信息）进行本地缓存，减少重复请求，提高响应速度。
